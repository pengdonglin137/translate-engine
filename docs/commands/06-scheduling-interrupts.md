# Linux Kernel - 中断调度

## 中断类型
- 硬中断：来自硬件设备的异步信号
- 软中断：延迟执行的中断下半部
- NMI：不可屏蔽中断，用于严重错误

## 中断处理流程
1. 硬件触发中断
2. CPU保存上下文，跳转到中断处理程序
3. 执行中断处理程序（上半部）
4. 调度软中断或任务队列（下半部）
5. 恢复被中断的进程

## 中断下半部机制
- 软中断（softirq）：静态定义，适合高频中断
- tasklet：基于软中断，动态创建
- 工作队列（workqueue）：可睡眠，适合长时间操作

## 中断亲和性
- 通过/proc/irq/<irq>/smp_affinity设置
- 控制中断在哪个CPU上处理
- 对性能优化很重要

## 中断负载均衡
- 内核自动在CPU间分发中断
- 可通过irqbalance工具优化
- NUMA系统中考虑节点局部性
