# 信雅达翻译标准

## 定义

**信雅达**源自严复的翻译理论，是评价翻译质量的三大标准：

| 维度 | 英文 | 定义 | 评判标准 |
|------|------|------|---------|
| **信** | Faithfulness | 忠实原文含义 | 不误译、不遗漏、不添加 |
| **雅** | Elegance | 优雅的中文表达 | 无翻译腔、符合中文写作习惯 |
| **达** | Expressiveness | 清晰流畅传达 | 逻辑清晰、句式自然、易于理解 |

## 信 (Faithfulness)

### 要求
- 准确传达原文的技术含义
- 不遗漏重要信息
- 不添加原文没有的内容
- 保持原文的语气和风格（幽默、严肃、反讽等）

### 常见问题
1. **误译**: 将 "keypunch" 译为 "困难" 而非 "打孔机"
2. **遗漏**: 跳过整段或整句未翻译
3. **添加**: 添加原文没有的强调或解释
4. **语气丢失**: 将幽默的原文译为严肃的中文

## 雅 (Elegance)

### 要求
- 使用自然流畅的中文
- 避免翻译腔（Chinglish）
- 符合中文技术写作规范
- 术语使用一致

### 常见翻译腔
1. **被动语态滥用**: "被...所持有" → 应改为主动语态
2. **的字堆叠**: "通信相对于处理的高overhead" → 应拆分
3. **过长定语**: "面向那些希望避免学习...以便更好地继续其并发编程生活的人" → 应拆分
4. **英文句式直译**: "这一巨大变化的产生基于两个基本原因" → 应为 "源于"

### 修复技巧
- 被动→主动: "被锁保护" → "锁保护了"
- 长句拆分: 80字以上的句子考虑拆分
- 减少"的": 连续3个以上"的"需要重构
- 文化适配: 使用中文惯用表达

## 达 (Expressiveness)

### 要求
- 逻辑清晰，易于理解
- 技术解释准确明了
- 句式自然，符合中文阅读习惯
- 适当使用逻辑连接词

### 常见问题
1. **歧义**: "直觉的时序行为" 可能有两种理解
2. **过长复合句**: 嵌套多层从句，难以理解
3. **逻辑跳跃**: 缺少必要的过渡和连接
4. **术语堆砌**: 连续使用多个专业术语，没有解释

### 修复技巧
- 歧义→明确: 添加限定词或重构句子
- 长句→短句: 拆分为2-3个短句
- 补充连接: 添加 "因此"、"然而"、"此外" 等
- 术语注释: 首次出现时加括号说明

## 实用检查清单

### 翻译前
- [ ] 理解原文的技术背景
- [ ] 确认术语表中的翻译
- [ ] 了解原文的语气风格

### 翻译后
- [ ] 信: 是否忠实原文含义？
- [ ] 雅: 是否避免翻译腔？
- [ ] 达: 是否清晰易懂？
- [ ] 术语: 是否与术语表一致？
- [ ] 格式: LaTeX/Markdown 语法是否正确？
- [ ] 构建: 是否能成功生成 PDF/HTML？
